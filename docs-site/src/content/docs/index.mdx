## Why footprintjs?

<CardGrid stagger>
  <Card title="Self-documenting pipelines" icon="document">
    Every stage writes its own narrative as it runs — no post-processing, no
    log parsing. `chart.recorder(narrative()).run()` tells you exactly what
    happened and why.
  </Card>
  <Card title="Typed from the start" icon="approve-check-circle">
    Define your state as a TypeScript interface. `TypedScope<T>` gives you
    full type inference inside every stage — no casts, no `getValue() as string`.
  </Card>
  <Card title="Decision evidence capture" icon="magnifier">
    `decide()` and `select()` auto-capture which values matched which rules.
    Narrative: *"creditScore 750 gt 700 ✓ → chose approved."*
  </Card>
  <Card title="Composable by design" icon="puzzle">
    Subflows compose like functions. `getSubtreeSnapshot()` gives you
    time-travel debugging at any depth. Works with MCP and OpenAPI out of the box.
  </Card>
</CardGrid>

## The 3-step pattern

```ts
import { flowChart, decide, narrative } from 'footprintjs';

// 1. Define your state
interface LoanState {
  creditScore: number;
  dti: number;
  decision?: string;
}

// 2. Build your flowchart
const chart = flowChart<LoanState>('Intake', async (scope) => {
  scope.creditScore = 750;
  scope.dti = 0.38;
}, 'intake')
  .addDeciderFunction('ClassifyRisk', (scope) => {
    return decide(scope, [
      { when: { creditScore: { gt: 700 }, dti: { lt: 0.43 } }, then: 'approved', label: 'Good credit' },
    ], 'rejected');
  }, 'classify-risk')
  .addFunctionBranch('approved', 'Approve', async (scope) => {
    scope.decision = 'Approved';
  })
  .addFunctionBranch('rejected', 'Reject', async (scope) => {
    scope.decision = 'Rejected';
  })
  .setDefault('rejected')
  .end()
  .build();

// 3. Run and observe
const rec = narrative();
await chart.recorder(rec).run();

console.log(rec.lines().join('\n'));
// Stage 1: The process began with Intake.
// Stage 2: Next, it moved on to ClassifyRisk.
// [Condition]: It evaluated Rule 0 "Good credit": creditScore 750 gt 700 ✓, and chose approved.
// Stage 3: Next, it moved on to Approve.
```
