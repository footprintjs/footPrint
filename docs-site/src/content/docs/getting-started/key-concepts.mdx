## The flowchart pattern

A footprintjs pipeline is a **directed graph of stages**. Each stage is an async function that reads and writes to a shared typed scope. Stages are connected sequentially, or via branching (`addDeciderFunction`) or parallel fan-out (`addSelectorFunction`).

```
Intake ‚Üí ClassifyRisk ‚îÄ‚îÄ‚ñ∫ approved ‚Üí Approve
                      ‚îî‚îÄ‚îÄ‚ñ∫ rejected ‚Üí Reject
```

The graph is built once (`.build()`), then executed many times (`.run()`). The graph structure is immutable; only the scope state changes per run.

## TypedScope ‚Äî typed shared state

`TypedScope<T>` is a deep Proxy over your state interface. It tracks every read and write, enabling recorders to observe exactly what happened and when.

```ts
interface LoanState {
  creditScore: number;
  customer: { name: string; zip: string };
  tags: string[];
}

// Inside a stage:
scope.creditScore = 750;              // typed write
scope.customer.zip = '90210';         // deep write
scope.tags.push('vip');               // array copy-on-write
const score = scope.creditScore;      // tracked read
```

<Aside type="caution">
  Don't use `getValue()`/`setValue()` in TypedScope stages ‚Äî use direct property access. `$`-prefixed methods (`$break`, `$debug`, `$metric`) are escape hatches for side effects.
</Aside>

## Observer channels

footprintjs has three runtime observer channels that fire at different times:

| Channel | Interface | Fires when |
|---|---|---|
| **Scope recorder** | `ScopeRecorder` | During stage execution (reads, writes, commits) |
| **Flow recorder** | `FlowRecorder` | After stage execution (decisions, forks, loops) |
| **Emit recorder** | `EmitRecorder` | When a stage calls `scope.$emit(name, payload)` |

A single object can implement hooks across all three channels ‚Äî that union is a `CombinedRecorder`, attached with one `attachCombinedRecorder()` call. `narrative()` returns exactly such a recorder (the `CombinedNarrativeRecorder`): it merges scope, flow, and emit events into a single timeline. You attach it via `.recorder(narrative())`.

## Collect during traversal

<Aside type="tip">
  **Core principle:** All data collection ‚Äî narrative, metrics, manifest ‚Äî happens as side effects of the single execution pass. Never post-process or walk the graph again.
</Aside>

This is why recorders exist: not as a debugging afterthought, but as the primary way to observe pipeline behavior in production.
